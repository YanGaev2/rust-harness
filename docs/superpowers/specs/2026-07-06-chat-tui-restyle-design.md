# Chat TUI restyle — design (2026-07-06)

## Цель

Привести чат-TUI (`src/tui.rs`, рендер-слой `ChatTuiApp`) к минималистичному стилю
Claude Code и закрыть четыре жалобы:

1. UI выглядит несовременно (рамки вокруг всего, пёстрая палитра, цветной бейдж в шапке).
2. В транскрипте виден бейдж `✎ think` — само слово «think» не должно отображаться.
3. Каретка в поле ввода встаёт не туда (нет переноса длинных строк; при выходе за
   правую границу `set_cursor_position` вообще не вызывается и каретка «замерзает»).
4. Меню команд при вводе `/` не появляется у пользователя (код попапа есть и бинарник
   свежее исходников — нужен рендер под вводом + тест на содержимое кадра + живая проверка).

## Границы

- Меняется только рендер-слой и расчёт каретки в `src/tui.rs` (функции `render_chat_*`,
  `entry_styled_lines`, `render_completion_popup`) плюс связанные тесты.
- State machine (`ChatTuiApp::handle_key`, completion-логика, история, скролл) не меняется,
  кроме мест, где рендеру нужны новые данные (например, ширина для wrap-расчёта каретки).
- Setup-TUI (`render_setup_tui`, `render_tui`) не трогаем — отдельная задача.
- Архитектурный рефакторинг по `docs/tui-reference-analysis.md` — вне scope.

## Решения

### Палитра (единая, вместо текущего разнобоя)

- Акцент: Cyan — маркеры ассистента, выделение в меню команд, имя провайдера.
- Основной текст: default (белый терминала). Вторичный: DarkGray.
- Статусы тулов: Green (ok), Red (failed), DarkGray (running — вместо жёлтого).
- Yellow и Magenta из чата убираются: system-строки → DarkGray, memo → dim Cyan.
- Чёрная заливка фона (`bg(Color::Black)`) убирается — фон терминала пользователя.

### Компоновка (сверху вниз)

1. **Транскрипт** — без рамки и заголовка « Session », занимает всё пространство,
   слева отступ 1–2 колонки. Индикатор прокрутки: dim-строка `↑ N` в правом верхнем
   углу области, только когда scroll > 0. Шапка-заголовок удаляется: провайдер и
   workspace переезжают в статус-строку.
2. **Спиннер-строка** — при busy над полем ввода: `✻ Working… (Esc — прервать)`,
   dim; из транскрипта и шапки спиннер убирается.
3. **Поле ввода** — единственная рамка на экране: Rounded, постоянно DarkGray
   (фокус всегда в поле ввода, менять цвет не от чего — мигание не нужно).
   Без заголовка « Message ». Внутри: префикс `> ` (dim), паддинг 1 слева/справа,
   мягкий перенос длинных строк, высота растёт до 6 строк с учётом переноса.
4. **Меню команд** — рендерится ПОД полем ввода (вытесняет статус-строку, до 6 строк),
   а не поверх транскрипта: `/имя` (набранная часть подсвечена) + dim-описание,
   выбранная строка — фон Cyan/чёрный текст. Управление прежнее (↑↓, Tab, Esc).
5. **Статус-строка** — одна dim-строка внизу: слева `provider/model · workspace`,
   справа контекстные подсказки клавиш (как сейчас в футере).

### Транскрипт: рендер записей (`entry_styled_lines`)

- **User**: `> текст` — весь блок dim (как в Claude Code), продолжения строк с отступом 2.
- **Assistant**: маркер `●` (Cyan) на первой строке, markdown-строки с отступом 2.
  Бейдж `▌ ai` удаляется.
- **Thinking**: НИКАКОГО бейджа и слова «think». Просто dim italic текст с отступом 2.
  Plain-text проекция `entry_plain_lines` сохраняет префикс `thinking:` (нужен тестам
  и non-TTY выводу — на экран не попадает).
- **Tool**: `● file.read(args…)` — маркер цветом статуса (DarkGray/Green/Red), имя
  жирным, args dim; результат второй строкой `  ⎿ summary` dim. Строка
  `tool round N: …` из транскрипта убирается (шум) — событие остаётся, но рендерится
  только когда tool_calls > 1, как dim-строка.
- **System**: `· текст` DarkGray; `memo:` — dim Cyan.

### Каретка (баг «не по середине»)

Расчёт позиции переписывается под перенос: логическая строка режется по внутренней
ширине (ширина области − рамка − паддинг − `> ` на первой строке), row/col каретки
вычисляются по этим визуальным строкам. Paragraph получает те же развёрнутые строки
(ручной wrap), чтобы текст и каретка всегда совпадали. Каретка никогда не «замерзает»:
позиция клампится в границы внутренней области.

### Меню команд: диагностика «не появляется»

- TestBackend-тест: набрать `/` → в буфере кадра присутствуют `/model` и описание;
  набрать `/mo` → список сужается; Esc → меню исчезает.
- После пересборки — живая проверка `harness` в терминале. Если не воспроизводится
  в коде, значит проблема была в старом бинарнике/окружении пользователя.

## Тесты

- Обновить существующие TestBackend-снапшоты в `tests/tui_screen.rs` под новую разметку.
- Новые: (а) в кадре нет слова «think» при ChatEntry::Thinking; (б) каретка после
  длинной строки с переносом стоит в ожидаемой позиции; (в) меню команд видно в кадре
  под областью ввода; (г) статус-строка содержит провайдера.
- `cargo fmt`, `cargo clippy --all-targets -- -D warnings`, `cargo test` — зелёные.

## Ошибки/крайние случаи

- Узкий терминал: паддинги и `> ` учитываются через saturating-арифметику; меню
  команд обрезается по доступной высоте.
- Пустой ввод: placeholder dim italic, каретка сразу после `> `.
- Многострочный ввод + перенос: высота поля считается по визуальным строкам (cap 6).
